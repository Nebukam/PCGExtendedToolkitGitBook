# Plugin Settings

Global configuration for PCGEx. Found in **Project Settings > Plugins > PCGEx**.

These settings apply project-wide and affect default behaviors across all PCGEx nodes. Most can be left at their defaults â€” <mark style="color:$warning;">adjust them when you have a specific reason.</mark>

### Defaults

<details>

<summary><strong>World Up</strong> <code>FVector</code></summary>

The "up" vector used internally by PCGEx as the default Z direction.

Default: `(0, 0, 1)` (Unreal's default up)

</details>

<details>

<summary><strong>World Forward</strong> <code>FVector</code></summary>

The "forward" vector used internally by PCGEx as the default X direction. Should be set relative to the World Up axis.

Default: `(1, 0, 0)` (Unreal's default forward)

</details>

### Performance â€” Defaults

<details>

<summary><strong>Default Cache Node Output</strong> <code>bool</code></summary>

Default value applied to node output caching when a node's cache setting is set to `Default`.

Default: `true`

{% hint style="warning" %}
Some nodes may behave unexpectedly when working with cached data. Disable per-node if you encounter issues.
{% endhint %}

</details>

<details>

<summary><strong>Default Scoped Attribute Get</strong> <code>bool</code></summary>

Default value for scoped attribute access when set to `Default` on individual nodes.

Default: `true`

</details>

<details>

<summary><strong>Bulk Init Data</strong> <code>bool</code></summary>

Default value for bulk data initialization when set to `Default` on individual nodes.

Default: `false`

</details>

<details>

<summary><strong>Execution Policy</strong> <code>EPCGExExecutionPolicy</code></summary>

Default execution policy for PCGEx nodes (synchronous, async, etc.).

Default: `Default`

</details>

### Performance â€” Cluster

<details>

<summary><strong>Use Delaunator</strong> <code>bool</code></summary>

Use the Delaunator library for Delaunay triangulation. Faster than the Unreal Geometry Script alternative.

Default: `true`

</details>

<details>

<summary><strong>Small Cluster Size</strong> <code>int32</code></summary>

Threshold below which a cluster is considered "small" â€” may affect internal processing strategies.

Default: `512`

Min: `1`

</details>

<details>

<summary><strong>Cluster Default Batch Chunk Size</strong> <code>int32</code></summary>

Default chunk size for batched cluster processing. Controls how work is divided across threads.

Default: `512`

Min: `1`

</details>

<details>

<summary><strong>Default Scoped Index Lookup Build</strong> <code>bool</code></summary>

Default value for scoped index lookup building on cluster operations.

Default: `false`

</details>

<details>

<summary><strong>Cache Clusters</strong> <code>bool</code></summary>

Allow caching of cluster data so it doesn't have to be rebuilt every time data passes between nodes.

Default: `true`

</details>

<details>

<summary><strong>Default Build And Cache Clusters</strong> <code>bool</code></summary>

Default value for new nodes' "Build and Cache Clusters" setting (editable per-node in Graph Output Settings).

Default: `true`

ðŸ“‹ _Visible when Cache Clusters is enabled_

</details>

### Debug

<details>

<summary><strong>Persistent Debug</strong> <code>bool</code></summary>

If enabled, debug visuals generated by PCG will not be transient (pre-5.6 behavior). Requires restarting the editor.

Default: `false`

</details>

<details>

<summary><strong>Assert On Empty Thread</strong> <code>bool</code></summary>

If enabled, an assertion is raised when attempting to schedule a zero-work task. Requires a debugger attached to the editor â€” otherwise the editor will crash.

Default: `false`

</details>

### Colors and Semantics

Visual customization for the PCG graph editor. These control node title colors and pin styling.

<details>

<summary><strong>Tone Down Optional Pins</strong> <code>bool</code></summary>

When enabled, non-required pins that are disconnected are visually toned down. Reduces confusion about which pins matter for a given configuration.

Default: `true`

</details>

<details>

<summary><strong>Use Native Colors If Possible</strong> <code>bool</code></summary>

When enabled, PCGEx nodes use vanilla PCG color semantics where applicable (filters, spawners, etc.) to stay visually consistent with the built-in node palette.

Default: `true`

</details>
